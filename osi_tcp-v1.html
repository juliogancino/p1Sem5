📜 1. Contexto Histórico y Necesidad

Problema inicial (años 70)

Cada fabricante (IBM, DEC, Xerox) implementaba su propia pila de protocolos. La incompatibilidad impedía la comunicación entre sistemas heterogéneos.

Solución: Modelos de referencia que estandarizan las funciones de comunicación.

ISO y ARPA

ISO (International Organization for Standardization) inició en 1977 el desarrollo del modelo OSI, publicado en 1984 como ISO/IEC 7498.

DARPA (Defense Advanced Research Projects Agency) financió el desarrollo de TCP/IP a finales de los 70, base de ARPANET.

Dato real: El 1 de enero de 1983, ARPANET migró oficialmente a TCP/IP, marcando el nacimiento de Internet tal como lo conocemos.

🧩 2. ¿Qué es un Modelo de Referencia?

Un modelo de referencia es una división estructurada en capas que describe las funciones necesarias para la comunicación entre sistemas. Cada capa ofrece servicios a la capa superior y utiliza servicios de la capa inferior.

Ventajas de las capas

  • ✅ Modularidad y desarrollo independiente
  • ✅ Interoperabilidad entre fabricantes
  • ✅ Facilita la enseñanza y solución de problemas
  • ✅ Cambios en una capa no afectan a las demás

Analogía del envío postal

📝 Carta (datos) → ✉️ Sobre (encabezado) → 🏢 Oficina postal (enrutamiento) → 🚚 Transporte (medio físico).

Cada nivel tiene su propia responsabilidad y no necesita conocer los detalles internos de los otros.

🏛️ 3. Modelo OSI – Visión General

El modelo Open Systems Interconnection consta de 7 capas. Fue desarrollado por ISO y estandarizado en 1984 (ISO/IEC 7498-1). Es un modelo teórico que sirve como referencia para entender la comunicación en redes.

7. AplicaciónHTTP, FTP, SMTP, DNS, Telnet
6. PresentaciónSSL/TLS, MIME, JPEG, ASCII
5. SesiónNetBIOS, PPTP, RPC, Sockets
4. TransporteTCP, UDP, SCTP
3. RedIP, ICMP, OSPF, BGP, IPsec
2. Enlace de datosEthernet, PPP, MAC/LLC, ARP
1. FísicaRS-232, 1000BASE-T, fibra óptica

Nemotecnia en inglés: "Please Do Not Throw Sausage Pizza Away" (Physical, Data Link, Network, Transport, Session, Presentation, Application).

🔍 4. OSI Capas 1 a 4 – Detalle Técnico

Capa 1 – Física

Transmisión de bits a través del medio físico. Define voltajes, conectores, frecuencias, modulación. Ej: IEEE 802.3 (Ethernet), fibra óptica, cable coaxial, RJ45.

Unidad de datos: bit.

Equipos: hubs, repetidores, cables, conectores.

Capa 2 – Enlace de datos

Control de acceso al medio (MAC) y detección de errores (CRC). Subcapas LLC y MAC. Direcciones físicas (MAC address de 48 bits).

Unidad: trama (frame).

Protocolos: Ethernet, PPP, HDLC, Frame Relay.

Equipos: switches, bridges.

Capa 3 – Red

Enrutamiento y direccionamiento lógico (IP). Determina la mejor ruta entre origen y destino. Control de congestión.

Unidad: paquete.

Protocolos: IPv4, IPv6, ICMP, OSPF, BGP, RIP.

Equipos: routers, capa 3 switches.

Capa 4 – Transporte

Comunicación extremo a extremo confiable. Control de flujo, segmentación, corrección de errores, control de congestión.

Unidad: segmento (TCP) / datagrama (UDP).

Protocolos: TCP (conexión, confiable), UDP (sin conexión, rápido), SCTP.

🔍 5. OSI Capas 5 a 7 – Detalle Técnico

Capa 5 – Sesión

Gestiona el diálogo entre aplicaciones. Establece, mantiene y finaliza sesiones. Sincronización y puntos de control para recuperación.

Protocolos: NetBIOS, PPTP, RPC (Remote Procedure Call), Sockets de Berkeley.

Capa 6 – Presentación

"Traductor" de formatos. Compresión, cifrado y conversión de datos (ASCII, EBCDIC, JPEG, MPEG, GIF). SSL/TLS opera en esta capa.

Garantiza que los datos enviados por la capa de aplicación de un sistema sean legibles por la capa de aplicación de otro.

Capa 7 – Aplicación

Interfaz con el usuario y servicios de red. No es la aplicación en sí, sino los protocolos que esta utiliza para comunicarse.

Protocolos: HTTP (Web), FTP (archivos), SMTP (correo), DNS (nombres), Telnet, SNMP, DHCP.

Dato importante: En la práctica, las capas 5 (Sesión) y 6 (Presentación) a menudo se implementan dentro de la aplicación, lo que llevó al modelo TCP/IP a fusionarlas en una sola capa de Aplicación.

🌍 6. Modelo TCP/IP – Visión General

Desarrollado por DARPA y documentado formalmente en RFC 1122 (1989). Consta de 4 capas y es el modelo que realmente impulsa Internet. A diferencia de OSI, TCP/IP se desarrolló a partir de protocolos existentes.

4. AplicaciónHTTP, FTP, DNS, SMTP, SSH, DHCP
3. TransporteTCP, UDP
2. InternetIP (IPv4, IPv6), ICMP, ARP, IGMP
1. Acceso a la RedEthernet, Wi-Fi (802.11), PPP, DSL

RFC 1122: "Requirements for Internet Hosts – Communication Layers" define formalmente la arquitectura TCP/IP y los requisitos de cada capa.

📡 7. TCP/IP – Detalle de Capas

Capa de Aplicación

Fusiona las capas 5-6-7 de OSI. Provee servicios directamente a las aplicaciones. Protocolos: HTTP (web, puerto 80), HTTPS (443), SMTP (25), FTP (21), DNS (53), DHCP, Telnet, SSH (22).

Capa de Transporte

Similar a OSI capa 4. TCP: confiable, orientado a conexión, control de congestión, retransmisión. UDP: rápido, sin conexión, para streaming, VoIP y DNS.

Capa de Internet

Corresponde a la capa 3 de OSI. Encamina paquetes a través de redes interconectadas. IPv4: direcciones de 32 bits (4.3 mil millones). IPv6: 128 bits (340 sextillones). ICMP para diagnóstico (ping, traceroute).

Capa de Acceso a la Red

Agrupa capas 1 y 2 de OSI. Maneja el hardware, direcciones MAC, acceso al medio físico. Ethernet (IEEE 802.3), Wi-Fi (IEEE 802.11), PPP, DOCSIS (cable).

⚖️ 8. Comparación Visual OSI vs TCP/IP

OSI: modelo teórico/educativo. Separa claramente sesión, presentación y aplicación.

TCP/IP: modelo práctico. Agrupa las capas superiores en una sola "Aplicación" y las inferiores en "Acceso a la Red".

Ambos comparten las capas de Transporte y Red (Internet) con funciones muy similares.

📦 10. Encapsulación de Datos

En el origen, los datos descienden por las capas y cada una añade su encabezado (header) con información de control. En la capa de enlace se añade también un tráiler para detección de errores. En el destino se invierte el proceso (desencapsulación).

Unidad de datos por capa: Datos (App) → Segmento (TCP) o Datagrama (UDP) → Paquete (IP) → Trama (Ethernet) → Bits (Físico).

  1. Datos (Capas de Aplicación/Presentación/Sesión): Se crean los datos originales.
  2. Segmento (Capa de Transporte): Se recibe la información y se le agrega una cabecera (TCP o UDP) para gestionar la comunicación (puertos de origen/destino).
  3. Paquete (Capa de Red): Se encapsula el segmento añadiendo una cabecera con direcciones lógicas (direcciones IP de origen y destino).
  4. Trama (Capa de Enlace de Datos): Se añade una cabecera física (MAC address) y un verificador de errores (CRC) al final para asegurar la integridad de la trama en la LAN.
  5. Bits (Capa Física): La trama se convierte en una serie de bits (señales eléctricas, luz o radio) para ser transmitida por el medio.

📍 11. Direccionamiento en las Capas

Direcciones Físicas (Capa 2)

MAC address: 48 bits (6 bytes), única por NIC. Ej: 00:1A:2B:3C:4D:5E. Utilizada en Ethernet, Wi-Fi. Asignada por el fabricante.

Direcciones Lógicas (Capa 3)

IPv4: 32 bits (ej. 192.168.1.10). IPv6: 128 bits (ej. 2001:db8::1). Jerárquicas y enrutables a nivel global.

Puertos (Capa 4)

Identifican procesos/aplicaciones. TCP/UDP: números de 16 bits (0-65535). Ej: HTTP → 80, HTTPS → 443, DNS → 53, SSH → 22.

El protocolo ARP (Address Resolution Protocol) traduce direcciones IP a direcciones MAC mediante broadcast en la capa de enlace.
